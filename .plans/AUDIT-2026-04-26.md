## Critical Findings

### C1 — API Key Prefix/Suffix Leaked to stdout
- **File:** `run_agent.py:1139`
- **Issue:** Prints `Using token: {key[:8]}...{key[-4:]}` — exposes first 8 and last 4 chars of API key to stdout, logs, and downstream consumers (Telegram, Discord). For `sk-ant-` keys, this narrows brute-force space significantly.
- **Fix:** Remove the print entirely or use full masking (`***...***`).

### C2 — Shell Injection via Unsanitized container_id
- **File:** `tools/environments/docker.py:562-575`
- **Issue:** `_container_id` interpolated into `shell=True` subprocess call: `f"(timeout 60 {self._docker_exe} stop {self._container_id} || ..."`. Attacker-controlled Docker API response or config injection → arbitrary command execution.
- **Fix:** Use `shlex.quote()` or switch to list-form subprocess without `shell=True`.

### C3 — Shell Injection via TUI shell.exec
- **File:** `tui_gateway/handlers/_misc.py:422-423`
- **Issue:** `shell.exec` JSON-RPC passes user-supplied command directly to `subprocess.run(cmd, shell=True)`. Blocklist-only defense (`detect_dangerous_command`) does not cover all injection vectors.
- **Fix:** Remove `shell=True`, use list-form execution.

### C4 — Shell Injection via Quick Commands
- **File:** `tui_gateway/handlers/_tools.py:257-259`, `cli.py:6179-6180`
- **Issue:** Config-sourced quick commands executed with `shell=True`. Compromised `config.yaml` or plugin = RCE.
- **Fix:** Use `shlex.split()` + list-form, or validate command source.

### C5 — File Descriptor Leak on Every Terminal Command
- **File:** `tools/environments/local.py:393`
- **Issue:** `cwd_path = open(self._cwd_file).read().strip()` — file handle never closed. Called on every terminal command execution via `_update_cwd()`.
- **Fix:** Use `with open(self._cwd_file) as f: cwd_path = f.read().strip()`.

### C6 — Module-Level File Handle Never Closed
- **File:** `tools/mcp_tool.py:131,138`
- **Issue:** `_mcp_stderr_log_fh` opened once and stored as module global. No `atexit` handler. Log rotation keeps stale fd open.
- **Fix:** Add `atexit.register(_mcp_stderr_log_fh.close)` or use `with` pattern.

### C7 — TELEGRAM_ALLOWED_USERS Defaults to Allow-All
- **File:** `gateway/platforms/telegram.py:256-262`
- **Issue:** When `TELEGRAM_ALLOWED_USERS` is empty or unset, `_is_callback_user_authorized` returns `True`. Any Telegram user can click inline approval buttons and authorize dangerous commands.
- **Fix:** Default to deny-all. Require explicit user ID configuration.

### C8 — Memory Not Isolated Per User
- **File:** `tools/memory_tool.py:53-55,124-130`
- **Issue:** `get_memory_dir()` returns `$HERMES_HOME/memories/` — single global directory. In multi-user gateway deployments, all users share `MEMORY.md` and `USER.md`. User A's preferences are visible to and overwritable by User B.
- **Fix:** Scope memory directory by user_id when available.

### C9 — God Class: AIAgent (12,192 lines)
- **File:** `run_agent.py` (entire)
- **Issue:** 148 methods, 68 constructor params. `run_conversation` alone is 3,314 lines. Mixes API client lifecycle, provider routing, streaming orchestration, credential refresh, context compression, tool dispatch, session persistence. Any change to any of these concerns requires editing this single class.
- **Fix:** Decompose into focused services (see Architecture section).

### C10 — God Class: HermesCLI (11,104 lines)
- **File:** `cli.py` (entire)
- **Issue:** 96 methods mixing TUI rendering, agent lifecycle, command dispatch, file attachments, voice mode, approval workflows.
- **Fix:** Decompose along responsibility boundaries.

### C11 — God Class: GatewayRunner (11,170 lines)
- **File:** `gateway/run.py` (entire)
- **Issue:** 159 methods mixing platform lifecycle, session routing, voice/TTS warmup, memory flush, agent spawning, restart/drain, media delivery, authorization.
- **Fix:** Decompose along responsibility boundaries.

## High Findings

### H1 — .env File World-Readable
- **File:** `.env` (permissions 0644)
- **Issue:** `_secure_file(0600)` only called on save, not initial creation. Any system user can read secrets.
- **Fix:** `chmod 600 .env` and audit all file creation sites.

### H2 — Webhook Secret Printed to stdout
- **File:** `hermes_cli/webhook.py:178`
- **Issue:** `print(f"  Secret: {secret}")` prints full HMAC secret.
- **Fix:** Mask or remove.

### H3 — Client Secret Partially Exposed
- **File:** `hermes_cli/dingtalk_auth.py:292`
- **Issue:** First 8 chars of DingTalk client secret shown in output.
- **Fix:** Show only last 4 chars or full mask.

### H4 — SUDO_PASSWORD Held in Plaintext Memory
- **File:** `tools/terminal_tool.py:711-721,717`
- **Issue:** Password read from env, cached in module-level global `_cached_sudo_password`, piped to sudo via stdin. Present in `/proc/<pid>/environ` and crash dumps.
- **Fix:** Minimize lifetime, clear after use, consider keychain integration.

### H5 — No Per-User Inbound Rate Limiting
- **File:** `gateway/platforms/telegram.py:2386-2400`
- **Issue:** Single user can flood bot with messages, consuming all 8 concurrent agent slots, denying service to others.
- **Fix:** Per-chat_id rate limiter (token bucket).

### H6 — Deprecated ParseMode.MARKDOWN
- **File:** `gateway/platforms/telegram.py:1234,1360,1469,1503,1539,1585,1657,1689`
- **Issue:** Legacy Markdown mode deprecated by Telegram. Silent rendering failures with user-controlled text.
- **Fix:** Migrate to `MARKDOWN_V2` or `HTML`.

### H7 — Unbounded In-Memory Session Dict
- **File:** `gateway/session.py:560-562`
- **Issue:** `_entries` grows without bound. `prune_old_entries()` requires explicit invocation.
- **Fix:** Auto-prune on every Nth access or after a time threshold.

### H8 — No Content Validation for Audio/Video Cache
- **File:** `gateway/platforms/base.py:470-485`
- **Issue:** `cache_audio_from_bytes()` and `cache_video_from_bytes()` write arbitrary bytes without content check. Image cache validates via `_looks_like_image()` but audio/video do not.
- **Fix:** Add magic-byte validation for audio/video formats.

### H9 — TTS Temp Files Not Cleaned Up
- **File:** `tools/tts_tool.py:1049-1061`
- **Issue:** Original MP3/WAV files left on disk after Opus conversion. Orphaned files accumulate.
- **Fix:** Delete source file after successful conversion.

### H10 — Subprocess Log File Leaks
- **File:** `tools/rl_training_tool.py:336,359,400`
- **Issue:** `open()` without `with` for subprocess stdout/stderr redirection. Leaked on crash paths.
- **Fix:** Use `with open(...)` pattern.

### H11 — No HTTP Connection Pooling
- **File:** `tools/send_message_tool.py`, `tools/homeassistant_tool.py`
- **Issue:** New `aiohttp.ClientSession()` per request — new TCP+TLS per message.
- **Fix:** Shared session with connection pool.

### H12 — Unbounded Sticker Cache
- **File:** `gateway/sticker_cache.py`
- **Issue:** Loads entire JSON into memory, appends, rewrites. No eviction.
- **Fix:** Add LRU eviction with size cap.

### H13 — Class-Level Mutable State on GatewayRunner
- **File:** `gateway/run.py:576`
- **Issue:** `_running_agents_ts`, `_busy_input_mode`, etc. defined as class attributes shared across instances. Tests or multi-instance scenarios see shared state corruption.
- **Fix:** Move to instance attributes in `__init__`.

### H14 — Transport Abstraction Built But Not Adopted
- **File:** `agent/transports/` vs inline in `AIAgent`
- **Issue:** Clean `ProviderTransport` ABC exists but 59 inline provider checks remain in `AIAgent`. Wasted investment + duplicated logic.
- **Fix:** Migrate provider routing through transport abstraction.

### H15 — Bidirectional Coupling agent/ <-> hermes_cli/
- **File:** `agent/account_usage.py`, `agent/credential_pool.py`, `agent/auxiliary_client.py`
- **Issue:** agent/ imports from hermes_cli/ for auth and config. No clean layer boundary.
- **Fix:** Extract shared infra into `hermes_core/` package.

### H16 — Scattered Configuration
- **Issue:** 956 `os.environ[]` + 732 `os.getenv()` + ~120 YAML keys + 4 config files totaling 12K lines. No single source of truth.
- **Fix:** Centralize config access through typed config objects.

### H17 — 18+ OpenAI() Client Construction Sites
- **Issue:** No factory, no shared connection pool, no centralized retry/timeout.
- **Fix:** Introduce `LLMClientFactory` with shared sessions.

### H18 — Silent Error Swallowing (70+ sites)
- **File:** `cli.py` (70+ sites), `tools/mcp_tool.py` (6 sites), `tools/voice_mode.py` (7 sites), `tools/file_tools.py` (6 sites)
- **Issue:** Broad `except Exception: pass` blocks hide failures.
- **Fix:** At minimum log at warning level; narrow exception types.

## EDGY Mode Deep-Dive

### Architecture Overview

EDGY mode is a dual-persona system toggled via Telegram `/edgy` command, switching between:
- **Friday** (default): powered by MiniMax/other cloud LLMs, standard TTS, text responses
- **Sydney** (EDGY): powered by local Qwen3.6-35B-Heretic on oMLX, Chatterbox Turbo voice clone, voice-only responses, ComfyUI image generation with Sydney LoRA

### Component Map

```
User sends "/edgy" in Telegram
       |
       v
gateway/run.py:_handle_edgy_command() (L5929)
  -> set_edgy_session()         [edgy_state.py: global dicts]
  -> voice_mode = "voice_only"  [run.py: instance dict]
  -> model_override = Qwen3.6   [run.py: instance dict, HARDCODED]
  -> evict_cached_agent()       [run.py: cache mutation]
  -> set_current_thread_edgy()  [edgy_state.py: thread-local]
       |
       v
gateway/run.py:_run_agent() (L9900)
  -> reads is_edgy_session()    [edgy_state.py: global dict]
  -> set_current_thread_edgy()  [edgy_state.py: thread-local propagation]
  -> cache invalidation check   [run.py: monkey-patched _edgy_mode attr]
  -> creates AIAgent with model override
       |
       v
agent/message_builder.py (L56-71) OR run_agent.py (L4078-4098)
  -> is_current_thread_edgy()?  [edgy_state.py: thread-local READ]
  -> loads SYDNEY.md instead of SOUL.md
       |
       v
Qwen3.6-Heretic generates response (localhost:8000)
  -> may call image_generate tool -> ComfyUI (localhost:8188) + Sydney LoRA
       |
       v
gateway/run.py response delivery (L4806)
  -> voice_only mode suppresses text
  -> TTS: chatterbox provider -> sydney_tts_server.py (localhost:9001)
  -> Voice chunking: _chunkify_voice_text() with Sydney-specific hooks
  -> Audio delivered as Telegram voice bubbles (Opus .ogg)
```

### EDGY-Specific Findings

#### E1 — Thread-Local Leakage [CRITICAL]
- **File:** `gateway/edgy_state.py:22`, `gateway/run.py:9956,10439,10531`
- **Issue:** Thread-local `_local.is_edgy` is set at request start and cleared at 3 separate exit points. If any new error path is added without calling `set_current_thread_edgy(False)`, the next request on that worker thread gets Sydney's persona **silently**. No warning, no log.
- **Impact:** User sends message to Friday, gets Sydney's personality and voice with zero indication of the bug.

#### E2 — Duplicated Persona Loading [HIGH]
- **Files:** `agent/message_builder.py:56-71` AND `run_agent.py:4078-4098`
- **Issue:** Identical SYDNEY.md/SOUL.md loading logic copy-pasted in two files. Bug fixes to one copy are not automatically applied to the other.
- **Impact:** Behavioral divergence between gateway and CLI entry points.

#### E3 — Duplicated TTS Function [HIGH]
- **Files:** `tools/tts/providers.py:264-282` AND `tools/tts_tool.py:553-590`
- **Issue:** `_generate_chatterbox_tts()` exists in two files with different default model names (`DEFAULT_CHATTERBOX_MODEL` vs `"chatterbox-turbo"`).
- **Impact:** Inconsistent behavior depending on which code path is hit.

#### E4 — Hardcoded Model Config [HIGH]
- **File:** `gateway/run.py:5946-5952`
- **Issue:** Model name, port, provider, API key hardcoded in Python:
  ```python
  "model": "Qwen3.6-35B-A3B-Heretic-MLX-mxfp8",
  "base_url": "http://127.0.0.1:8000/v1",
  "api_key": "local-no-key",
  ```
- **Impact:** Changing model quant or LLM server requires editing an 11,170-line file.

#### E5 — No Health Check at Toggle [HIGH]
- **File:** `gateway/run.py:5929-5964`
- **Issue:** `/edgy` sets model override to localhost:8000 without checking if oMLX server is running. User sees "Edgy mode activated" but first message silently fails.
- **Impact:** Confusing UX. User thinks Sydney is active but gets error or timeout.

#### E6 — Voice Chunking Not Gated by Persona [MEDIUM]
- **File:** `gateway/run.py:6384-6410`
- **Issue:** `_chunkify_voice_text()` uses Sydney-specific hook keywords (`daddy`, `sir`, `johnny`, `want more`) but is not gated by edgy flag. Any TTS user gets chunked on these keywords.
- **Impact:** Non-edgy users with TTS get unexpected chunking behavior.

#### E7 — Monkey-Patched Cache Attribute [MEDIUM]
- **File:** `gateway/run.py:10140`
- **Issue:** `agent._edgy_mode = _is_edgy` patches attribute at runtime. Breaks if `AIAgent` adds `__slots__` or a real `_edgy_mode` property.
- **Impact:** Cache invalidation breaks silently, serving wrong persona.

#### E8 — Five Non-Atomic State Mutations [HIGH]
- **File:** `gateway/run.py:5938-5958`
- **Issue:** `/edgy` performs 5 state changes with no atomicity guarantee. Process crash between steps 2-5 leaves partial state (edgy session marked, voice-only set, but thread-local stale).
- **Impact:** Inconsistent state requiring manual `/normal` reset.

#### E9 — Image Generation Not Persona-Aware [MEDIUM]
- **File:** `tools/image_generation_tool.py:842-1027`
- **Issue:** Tool description and behavior hardcoded for Sydney LoRA (`sw33ny` trigger word). Not gated by persona. Non-edgy image generation also uses Sydney LoRA.
- **Impact:** Wrong LoRA applied in non-edgy mode; cannot use different image styles per persona.

#### E10 — TTS Server Single Concurrency [MEDIUM]
- **File:** `~/.hermes/sydney/sydney_tts_server.py`
- **Issue:** `threading.Semaphore(1)` serializes all TTS. While necessary for Apple Silicon, no backpressure signaling — requests queue indefinitely.
- **Impact:** Long queue times under load with no feedback to user.

## Architecture & First-Principles Critique

### God Class Analysis

The three core files total **34,466 lines** (13% of entire source):

| File | Lines | Methods | Params | Key Issues |
|---|---|---|---|---|
| `run_agent.py` | 12,192 | 148 | 68 constructor | 8+ distinct responsibilities |
| `cli.py` | 11,104 | 96 | — | TUI + agent + commands + voice |
| `gateway/run.py` | 11,170 | 159 | — | Platforms + sessions + agent + TTS + memory |

### Coupling Analysis

```
607 deferred imports across 3 God classes:
  - run_agent.py:  144 deferred imports
  - cli.py:        240 deferred imports
  - gateway/run.py: 223 deferred imports

Bidirectional coupling:
  hermes_cli/ <---> agent/
  tools/ ---------> hermes_cli/ (layer violation)

20+ AIAgent instantiation sites with no factory
18+ OpenAI() client constructions with no shared pool
59 scattered provider detection sites
1,781 API key reference sites
```

### Sustainability Verdict: At Risk

- `AIAgent` is a single point of failure for any change to streaming, API routing, credentials, tool dispatch, session management, or context compression.
- 607 deferred imports indicate circular dependency pressure hidden at runtime.
- The `agent/transports/` abstraction was built but not adopted — wasted investment.

### Scalability Verdict: Constrained

- Each gateway session spawns a full `AIAgent` in-process with no isolation.
- Session state is file-based with file locks — cannot scale to multi-instance.
- No HTTP connection pooling across TTS, model metadata, and messaging.
- Monolithic design prevents horizontal scaling.

### Efficiency Verdict: Mixed

- Over-engineered: `agent/transports/` ABC built but unused.
- Under-engineered: no connection pooling, no client factory, no provider registry.
- `hermes_cli/` contains core infra (auth 3.5K lines, config 4K lines, models 2.7K lines) but is named as CLI.

## Notable Strengths

1. **Security tooling is strong:** `tools/skills_guard.py` (credential + prompt injection detection), `tools/credential_files.py` (path traversal + symlink protection), `tools/approval.py` (dangerous command blocklist)
2. **Gateway graceful shutdown** is well-engineered: drain → interrupt → kill → cleanup → marker
3. **Platform reconnection** with exponential backoff (30s→300s cap, 20 max attempts)
4. **Concurrent agent limiting** via `asyncio.Semaphore(8)`
5. **Session store** uses `threading.Lock()` consistently; SQLite outside lock
6. **Document cache** has proper path traversal protection
7. **Tool registry** (`tools/registry.py`) is clean self-registration with AST discovery
8. **Transport abstraction** (`agent/transports/`) is well-designed when used
9. **Test-to-source ratio** 0.91:1 with 123 test files
10. **`_mask_api_key_for_logs()`** properly masks keys in debug output
11. **Credential stripping** in MCP error messages prevents accidental exposure
12. **EDGY TTS server** has good text sanitization (CJK/emoji stripping), proper atexit cleanup, and bound to 127.0.0.1 only

1. **Persona System refactor** — eliminates 10 EDGY findings at once (E1-E8, plus maintainability/scalability)
